# 什么是大中台

业务中台（例如：搜索事业部）、数据中台（例如：数据分析）、技术中台（例如：web框架），由很多团队来支持开发。

中台不直接面向消费者，提供基础设施、内部工具、内部平台，来为业务团队（比如前端）赋能提效。

举个例子（阿里）：搜索事业部、数据平台事业部、数据分享事业部，为前台（零售电商事业群）提供服务。前端专注于业务发展，创新需求。

中台首先是组织机制先行（单独的部门负责中台，公司调整组织架构，物理拆分成为独立的中台部门），其次是业务机制（将公共能力下沉为服务，做好服务连接，赋能业务部门）。

下面就是一个例子，大中台中有很多团队提供服务能力。为了方便业务使用，降低连接陈本，应该提供统一对外接口（比如API）。

# 大中台小前台模式适用场景

对于初创公司，显然人力成本和技术积累过高，不合适。对于之后，高速发展的公司（比如转转、拼多多），越早起步越好。因为好的中台能力，将会让内部业务团队的创新更便捷，成本更低。
